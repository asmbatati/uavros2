# Simulators

`uav_gz_sim` dispatches to one of six simulator backends via `simulator:=<name>` on the top-level launch. Status, scope, and install steps per backend:

| Simulator | Status | UAVs supported | Arms supported | PX4 path |
|---|---|---|---|---|
| **gazebo** | Working | all x500 variants | three_dof, openmanip_x, panda, ur5 | PX4 SITL + MAVROS / uXRCE-DDS |
| **webots** | Working (base only) | x500 | (none in this pass) | PX4 `make px4_sitl webots` |
| **mujoco** | Working (x500 + Panda) | x500 | panda (sweet spot) | none — `sim_control_bridge` |
| **isaac** | Scaffolded | — | — | PegasusSimulator (community) |
| **pybullet** | Scaffolded | x500 (kinematic) | — | none — placeholder PID |
| **genesis** | Stub | — | — | none |

## gazebo

Default and most complete. Requires:

- Gazebo Harmonic (`gz-harmonic`)
- `ros-jazzy-ros-gz` + `ros-jazzy-ros-gz-bridge`
- PX4 SITL (built by `install.sh`)
- MAVROS (custom fork, built by `install.sh`)

Launch:
```bash
ros2 launch uav_gz_sim sim.launch.py simulator:=gazebo uav:=x500_d435 arm:=panda world:=tugbot_depot
```

The launch chain refactors today's `sim.launch.py` + `gz_sim.launch.py` + `mavros.launch.py` into `launch/simulators/gazebo.launch.py`. Sensor topics flow through `ros_gz_bridge` and are remapped to the canonical names declared in `config/topics/canonical_topics.yaml`.

## webots

Working for x500 base only. Arm integration is left for a follow-up because `ros2_control` plugins for Webots are fiddly.

Requires:
- Webots R2024a or newer
- `ros-jazzy-webots-ros2`
- PX4 SITL built with the `webots` simulator target: `cd $DEV_DIR/PX4-Autopilot && make px4_sitl webots`

Launch:
```bash
ros2 launch uav_gz_sim sim.launch.py simulator:=webots uav:=x500 world:=tugbot_depot
```

Asset format: PROTO (generated from URDF via `webots_ros2_importer`). Note the Webots/Jazzy integration is sensitive to Webots version drift; pin to R2024a.

## mujoco

Working for x500 + Panda. **No PX4** — `sim_control_bridge.py` owns rotor mixing + attitude PID and publishes MAVROS-shaped state topics so downstream code is identical.

Requires:
- `pip install mujoco`
- `mujoco_ros2_control` (currently source-only; see https://github.com/moveit/mujoco_ros2_control)
- `ros-jazzy-ros2-control` + `ros-jazzy-ros2-controllers`

Launch:
```bash
ros2 launch uav_gz_sim sim.launch.py simulator:=mujoco uav:=x500 arm:=panda
```

Asset format: MJCF (generated by `scripts/convert/urdf_to_mjcf.py` from the URDF source; hand-patched for `<actuator>` and contact params).

## isaac

**Scaffolded only.** `launch/simulators/isaac.launch.py` documents the install path but is not wired end-to-end.

Requires:
- NVIDIA Isaac Sim 4.x via Omniverse Launcher
- PegasusSimulator (community PX4 backend): https://github.com/PegasusSimulator/PegasusSimulator
- USD assets pre-generated by `scripts/convert/urdf_to_usd.py` (must be run inside Isaac's Python interpreter)

The launch file prints the install steps and exits. Wiring it up is tracked as a follow-up — see [ROADMAP](#roadmap) below.

## pybullet

**Scaffolded.** Loads URDF directly (PyBullet has a native loader). No PX4. The included `sim_control_bridge` PyBullet backend is a placeholder PID — sufficient for headless CI smoke tests, not for serious flight dynamics.

Requires:
- `pip install pybullet`

Launch:
```bash
ros2 launch uav_gz_sim sim.launch.py simulator:=pybullet uav:=x500
```

## genesis

**Stub.** `launch/simulators/genesis.launch.py` prints "not implemented" and a pointer here. The Genesis API is still moving; revisit when the project stabilizes.

## Choosing a simulator

| If you need… | Use |
|---|---|
| PX4 SITL + ROS 2 integration end-to-end | gazebo |
| Photorealistic rendering, RL workflows | isaac (once wired) |
| Fast contact-rich arm control, planning research | mujoco |
| Cross-autopilot sandbox | webots |
| Headless CI smoke tests | pybullet |
| Reinforcement learning, mass parallel rollouts | genesis (future) |
